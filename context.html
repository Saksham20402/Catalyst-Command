Context Brief

Share this file at the start of any new chat session. It tells Claude exactly which files exist, what each section contains, and how to navigate without reading everything.

How it works
Each file has a CATALYST:MANIFEST comment at the very top. Any new chat reads it in the first 20 lines and knows the full map — no scanning.
Context isolation
Backend chat? Load tickets.html + area filter. Designer? Load the spec file only — self-contained. New session? Load this file first, nothing else.
Source of truth
Ticket data lives only in tickets.html (CATALYST:TICKETS:DATA). Design tokens live only in design-system.html. Never duplicate between files.

Workflow

Task Recipes

Add or update a ticket
New work, status change, close out
1
Load tickets.html — read manifest, go to CATALYST:TICKETS:DATA
2
Edit the JSON array. Add new ticket or change "status" field. Bump "updated" date.
3
Sync index.html: update CATALYST:STATS counts + CATALYST:ACTIVITY list.
Focus session on one area
Backend, mobile, AI agents, or pitch deck
1
Load context.html (this file) — Claude reads the manifest at the top.
2
Load tickets.html — the manifest lists ticket IDs by area at the top.
3
Tell Claude: "Focus on area 'backend' only — CAT-007, CAT-008".
Designer handoff
Create a spec and hand it off
1
Load specs/_template.html as base. Load design-system.html for tokens.
2
Create specs/[name]-v1.html. Fill all SPEC: sections. Link the ticket ID.
3
Register in specs/index.html at CATALYST:SPECS:REGISTRY. Hand off the single file.
Update design system
New token, component, or pattern
1
Load design-system.html — read the manifest to find the exact section anchor.
2
Target the anchor, e.g. CATALYST:DS:COLORS. Add the new entry inside.
3
If a new token, also update the CORE TOKENS block in the design-system.html manifest.
Add a new workstream
New area beyond the current 4
1
Update tickets.html manifest: add new area value to TICKET SCHEMA and AREA TICKET GROUPS.
2
Add area card to index.html CATALYST:WORKSTREAMS. Choose an accent color.
3
Update the AREA filter <select> in tickets.html UI with the new option.
Transfer context to new chat
Starting fresh with full project awareness
1
Share context.html — Claude reads the CATALYST:MANIFEST comment at the top.
2
Share only the specific files needed for the task. Don't share everything.
3
Use a prompt template (see below) to tell Claude exactly what to do.

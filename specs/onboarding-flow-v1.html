Handoff

Interactions

Get Started button Tap Screen 1
Push Screen 2 (Account Setup) with a slide-from-right transition, 320ms ease-out.
Already have an account Tap Screen 1
Navigate to the Sign In flow (separate from onboarding). Replace the entire onboarding stack — no back button back to onboarding.
Back button Tap Screens 2–4
Pop the current screen. Slide-from-left, 280ms. Preserve all previously entered data — do not clear.
Continue button Tap Screen 2
Validate email format and password (min 8 chars). If invalid — shake the relevant field (3 oscillations, ±6pt, 300ms). If valid — push Screen 3.
Avatar placeholder Tap Screen 3
Open system photo picker (PHPickerViewController / Android Photo Picker). On selection, crop to 1:1 circle, compress to ≤200KB. Preview appears in the avatar circle.
Selection chip Tap Screen 4
Toggle selected state. Spring animation on border and background color, 200ms. No minimum or maximum selection required.
Open Catalyst button Tap Screen 5
Mark onboarding as complete (write flag to UserDefaults / AsyncStorage). Transition to the main dashboard with a cross-dissolve, 400ms. Never show onboarding again.
Any screen Swipe right Screens 2–4
Same as tapping Back. iOS: system edge swipe enabled. Android: predictive back gesture enabled.
Handoff

Accessibility

✓All tap targets minimum 44 × 44pt — buttons, chips, avatar, and back arrow all exceed this.
✓Headline #F0F0F0 on #121212 — contrast ratio 18.2:1 (AAA).
✓Muted text rgba(255,255,255,0.4) on #121212 — contrast ratio 4.6:1 (AA). Do not go below 40% white opacity.
✓Password field uses secureTextEntry / inputType="textPassword". Strength label also read aloud by VoiceOver ("Password strength: strong").
✓Selection chips have accessibilityRole="checkbox" and announce selected state on toggle.
✓Progress dots have an accessibilityLabel: "Step X of 5".
✓Avatar picker announces: "Profile photo, not set. Double-tap to choose a photo."
✓Screen 5 checkmark animation respects UIAccessibility.isReduceMotionEnabled — skips animation and shows immediately if enabled.
Handoff

Dev Notes

Onboarding state persistence
Track progress with a step index in local state. Each screen mounts with previously entered data pre-filled — don't clear on back. Write completion flag onboarding_complete: true to persistent storage only on Screen 5 "Open Catalyst" tap, never earlier.
Account creation timing
Call POST /auth/register on Screen 2 "Continue" tap, not on Screen 5. If the API call fails, show an inline error beneath the email field. Do not navigate to Screen 3 until the API call succeeds. Show a loading spinner in the Continue button during the request.
Avatar upload
Compress to JPEG, max 200KB, before upload. Upload to PUT /user/avatar asynchronously while user continues to Screen 4 — do not block progression. If upload fails silently, retry up to 3 times with exponential backoff.
Preferences payload
Send selected chip values as an array: PATCH /user/preferences { "intents": ["planning", "ai-dev"] }. Fire and forget on Screen 4 Continue tap — do not block navigation if this call fails.
Keyboard avoidance
Screens 2 and 3 must push content above the keyboard. Use KeyboardAvoidingView with behavior="padding" on iOS and android:windowSoftInputMode="adjustResize". The Continue button must always remain visible above the keyboard.
